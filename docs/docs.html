Key Features

Feature Description
Alert Management Create, acknowledge, snooze, assign, and resolve alerts with full lifecycle tracking
Incident Management Track incidents with responders, timeline, notes, and linked alerts
On-Call Schedules Flexible scheduling with rotations, participants, and schedule overrides
Escalation Policies Multi-level escalation rules with configurable targets and delays
Team Management Organize users into teams with role-based member management
Notifications Multi-channel notifications via Email, Slack, and Microsoft Teams
Webhooks Incoming and outgoing webhooks for integrations
Real-time Updates WebSocket support for live dashboard updates

Architecture

Pulsar follows a clean architecture pattern with clear separation of concerns:

Frontend (Svelte) --> API Gateway (Go/Gin) --> Services Layer --> PostgreSQL
                    |                            |
               WebSocket Hub <-- Background Workers (Escalation, Notifications)
                                       |
                         External Integrations (Email, Slack, Teams)

Tech Stack

  • Backend: Go 1.24, Gin framework, PostgreSQL 16
  • Frontend: SvelteKit 2.0, TypeScript, Tailwind CSS
  • Authentication: JWT with refresh tokens
  • Real-time: WebSocket via gorilla/websocket
  • API Docs: Swagger/OpenAPI 3.0
  • Deployment: Docker & Docker Compose
